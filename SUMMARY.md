## Выполненные задачи

Все запрошенные улучшения успешно реализованы:

### ✅ 1. Автосмена настроек при падении VPN

**Реализовано:**
- Улучшенный скрипт мониторинга `vpn-monitor-improved.sh`
- **Ключевое отличие**: при смене сайта маскировки ключи/пароли НЕ меняются
- Обновляется только dest/sni для REALITY и URL для Hysteria2
- Клиенту НЕ нужно обновлять подписку после смены маскировки

**Как работает:**
```bash
# При падении VLESS:
1. Система находит новый рабочий сайт из белого списка
2. Обновляет ТОЛЬКО dest и serverNames в конфиге
3. Сохраняет текущие privateKey и publicKey
4. Перезапускает Xray
5. Клиент продолжает работать со старым ключом
```

**Настроено:**
- Интервал мониторинга: 5 минут (было 3)
- Systemd timer: `OnUnitActiveSec=5min`
- Первая проверка через 1 минуту после загрузки

**Что проверяется:**
- Работа всех протоколов (VLESS, Hysteria2, Trojan, VMess, Shadowsocks, WireGuard)
- Доступность сайта маскировки
- Статус сервисов (xray, hysteria-server, wg-quick@wg0)

**При обнаружении проблем:**
- Автоматический поиск нового рабочего сайта
- Обновление конфигурации БЕЗ смены ключей
- Перезапуск сервисов
- Генерация обновленной подписки
- Логирование всех действий

**Критически важно:**
- UUID, Public Key, пароли остаются прежними
- Меняется только SNI/dest для маскировки
- Клиенту НЕ нужно обновлять ключи
- Подписка обновляется автоматически с новым SNI

## Технические детали

### Как работает сохранение ключей

**Старая версия (vpn-monitor.sh):**
```bash
update_vless_config() {
    # Генерируем НОВЫЕ ключи
    local keys_output=$(/usr/local/bin/xray x25519)
    local private_key=$(echo "$keys_output" | awk '/Private/{print $NF}')
    local public_key=$(echo "$keys_output" | awk '/Public/{print $NF}')
    
    # Обновляем конфиг с НОВЫМИ ключами
    jq --arg privkey "$private_key" \
       --arg pubkey "$public_key" \
       '.inbounds[0].streamSettings.realitySettings.privateKey = $privkey |
        .inbounds[0].streamSettings.realitySettings.publicKey = $pubkey'
}
# Проблема: клиенту нужно обновить Public Key!
```

**Новая версия (vpn-monitor-improved.sh):**
```bash
update_vless_config() {
    # Получаем ТЕКУЩИЕ ключи из конфига
    local data=$(get_vless_data)
    local current_private_key=$(echo "$data" | cut -d'|' -f5)
    local current_public_key=$(echo "$data" | cut -d'|' -f4)
    
    # Обновляем ТОЛЬКО dest и serverNames
    jq --arg dest "$dest" \
       --arg sni "$domain" \
       --arg privkey "$current_private_key" \
       --arg pubkey "$current_public_key" \
       '.inbounds[0].streamSettings.realitySettings.dest = $dest |
        .inbounds[0].streamSettings.realitySettings.serverNames = [$sni] |
        .inbounds[0].streamSettings.realitySettings.privateKey = $privkey |
        .inbounds[0].streamSettings.realitySettings.publicKey = $pubkey'
}
# Решение: ключи остаются прежними, клиент продолжает работать!
```

### Структура подписки

```
subscription.txt:
├── vless://UUID@IP:443?...&pbk=PUBLIC_KEY&sni=NEW_SNI#VLESS
├── hysteria2://PASSWORD@IP:36712?sni=NEW_SNI#Hysteria2
├── trojan://PASSWORD@IP:8444#Trojan
├── vmess://BASE64_CONFIG#VMess
├── ss://BASE64_METHOD_PASSWORD@IP:8388#Shadowsocks
└── (WireGuard использует отдельный .conf файл)
```

При смене маскировки меняется только `sni=NEW_SNI`, остальное остается прежним.
